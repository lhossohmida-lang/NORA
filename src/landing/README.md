# صفحة الهبوط السينمائية — أفنان (Video-Driven Landing)

صفحة هبوط فاخرة بأسلوب **Editorial / Fashion**، عمودها الفقري **فيديوهات متسلسلة**
كامل الشاشة تنتقل بينها بـ **cross-fade** سينمائي عبر تمرير الفأرة، النقر على النقاط
الجانبية، لوحة المفاتيح، أو السحب باللمس. عربية **RTL** بالكامل، بثيم الزمرّدي/الذهبي.

> معاينة الواجهة (إطار الـHero): [`public/videos/01-hero.webp`](../../public/videos/01-hero.webp)
> — أو شغّل المشروع وافتح `/` لرؤيتها حيّة بكل الانتقالات.

## التشغيل

```bash
npm install
npm run dev      # ثم افتح http://localhost:5173/
```

- `/`       → صفحة الهبوط السينمائية (`CinematicLanding`).
- `/shop`   → المتجر الكامل (الشبكة، السلة، التفاصيل).
- الـ CTA في كل مشهد تقود إلى `/shop`، والإضافة للسلة من مشهد المنتجات تُكتب على
  نفس مفتاح `localStorage` (`afnan_cart`) فتظهر مباشرةً في سلة المتجر.

```bash
npm run build    # بناء للإنتاج
```

## بنية الملفات

| الملف | الدور |
|------|------|
| `src/landing/scenes.config.js` | **مصدر الحقيقة الوحيد**: عناوين، أوصاف، مسارات الفيديو/الـposter، نصوص الأزرار والروابط. |
| `src/landing/SceneVideo.jsx` | طبقة فيديو واحدة كامل الشاشة + poster + تحميل كسول + تشغيل عند التنشيط فقط. |
| `src/landing/VideoScenes.jsx` | المحرّك: فهرس المشهد، الاتجاه، wheel/نقر/كيبورد/لمس، cross-fade، شريط التقدّم، النقاط، تشغيل/إيقاف. |
| `src/components/CinematicLanding.jsx` | صفحة `/`: تجلب المنتجات الحيّة، تربط الإضافة للسلة، وتضيف SEO (العنوان + JSON-LD). |
| `public/videos/0X-*.mp4` + `.webp` | مقاطع المشاهد وصور الـposter. |

## كيف تستبدل الفيديوهات أو النصوص

- **فيديو:** ضَع ملفاً جديداً في `public/videos/` بنفس الاسم (مثل `01-hero.mp4`) مع
  poster مطابق (`01-hero.webp`). **لا حاجة لتعديل أي كود.** الموصى به: H.264 mp4،
  ≤ 1080p، حلقة 8–15 ثانية، الحجم < 4MB، مع poster بصيغة WebP لتفادي الشاشة السوداء.
- **نصوص / ترتيب المشاهد:** عدّل `scenes.config.js` فقط — `eyebrow`، `title`
  (مصفوفة = سطر لكل عنصر يُكشف تباعاً)، `description`، `cta`، و`kind` الذي يحدّد
  تخطيط المحتوى: `hero | collections | craft | products | testimonials | cta`.

## التحكّم

| الإجراء | النتيجة |
|--------|--------|
| عجلة الفأرة | مشهد للأمام/الخلف (قفل ~850ms يمنع القفز) |
| النقاط الجانبية | قفز مباشر لأي مشهد |
| ↑ / ↓ / Space | السابق / التالي |
| ← / → | تالي / سابق (وعي بالـRTL) |
| Home / End | الأول / الأخير |
| السحب لأعلى/أسفل (لمس) | تالي / سابق |
| زر ⏯ (أسفل اليسار) | إيقاف/تشغيل فيديو الخلفية |

## الأداء وإمكانية الوصول

- **تحميل كسول:** يُحمَّل المشهد النشط والمجاور فقط؛ ويُجلب الفيديو التالي مسبقاً
  (`preload="auto"`)، وتُوقَف الفيديوهات غير المرئية لتوفير المعالج.
- **`prefers-reduced-motion`:** إيقاف التشغيل التلقائي وعرض الـposter بدلاً منه،
  وإلغاء حركات framer (عبر `<MotionConfig reducedMotion="user">` في `App.jsx`).
- **A11y:** نقاط بـ `aria-label`/`aria-current`، تنقّل كامل بلوحة المفاتيح، نصوص بديلة
  للـposter، تباين ذهبي/عاجي عالٍ، وزر تشغيل/إيقاف ظاهر.
- **بدون تمرير عمودي تقليدي:** الصفحة طبقة `fixed` كامل الشاشة (يُستعاد تمرير الجسم عند
  مغادرة الصفحة).
- **SEO:** عنوان الصفحة + بيانات منظّمة `Store` (JSON-LD)، مع وسوم Open Graph في
  `index.html`.

## ملاحظات

- المقاطع المشحونة حالياً **عناصر نائبة** تعيد استخدام لقطات المتجر الحقيقية (فيلم
  القندورة + لقطة البوتيك). استبدليها بفيديوهات مخصّصة للمشاهد عند توفّرها.
- المشروع JavaScript (Vite + JSX)، لذا الإعداد `scenes.config.js` بدل `.ts` — والبنية
  مطابقة تماماً لِما طُلب باسم `scenes.config.ts`.
